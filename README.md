## Architecture

к°Ғ featureлҠ” **3к°ңмқҳ л Ҳмқҙм–ҙ**лЎң л¶„лҰ¬лҗ©лӢҲлӢӨ.

```
Presentation  вҶ’  Domain  вҶҗ  Data
(Riverpod)       (Models     (Firebase /
                  Repos)      API)
```

- **domain** вҖ” мҲңмҲҳ Dart. мҷёл¶Җ мқҳмЎҙм„ұ м—ҶмқҢ. л№„мҰҲлӢҲмҠӨ лЎңм§Ғмқҳ кі„м•Ҫм„ң.
- **data** вҖ” Firebase/API кө¬нҳ„мІҙ. domainмқҳ repository мқён„°нҺҳмқҙмҠӨлҘј кө¬нҳ„.
- **presentation** вҖ” UI + Riverpod provider. domainмқ„ нҶөн•ҙм„ңл§Ң dataм—җ м ‘к·ј.

## Getting Started

### 1. нҢЁнӮӨм§Җ м„Өм№ҳ

```bash
flutter pub get
```

### 2. Firebase м—°кІ°

```bash
# FlutterFire CLI м„Өм№ҳ
dart pub global activate flutterfire_cli

# Firebase н”„лЎңм қнҠё м—°кІ° (firebase_options.dart мһҗлҸҷ мғқм„ұ)
flutterfire configure
```

### 3. мҪ”л“ң мғқм„ұ

Riverpod generator, freezed, json_serializable мӮ¬мҡ© мӢң н•„мҲҳ.

```bash
dart run build_runner build --delete-conflicting-outputs
```

нҢҢмқј ліҖкІҪ мӢң watch лӘЁл“ң:

```bash
dart run build_runner watch --delete-conflicting-outputs
```

### 4. мӢӨн–ү

```bash
flutter run
```
